## 第二部分：核心技术解析

### 幻灯片 5：向量数据库基础
**什么是向量数据库？**
- 存储高维向量的专用数据库
- 支持高效的近似最近邻搜索（ANN）
- 典型技术：Milvus、Pinecone、Weaviate、FAISS

**核心能力**：
- 向量索引：HNSW、IVF等算法
- 相似度计算：余弦相似度、欧氏距离
- 实时检索：毫秒级响应

**在本项目中的应用**：
- 语义缓存模块：使用sentence-transformers生成文本嵌入
- 相似问答快速召回，避免重复计算

**视觉元素**：
- 向量空间示意图
- ANN搜索算法图解

### 幻灯片 6：语义搜索原理
**语义搜索 vs 关键词搜索**：

| 维度 | 关键词搜索 | 语义搜索 |
|------|-----------|---------|
| 匹配方式 | 精确匹配 | 语义相似度 |
| 理解能力 | 字面意义 | 深层含义 |
| 查询灵活性 | 需精确关键词 | 自然语言 |
| 典型场景 | 文档检索 | 智能问答 |

**技术实现**：
- 文本嵌入（Embedding）：将文本转换为向量表示
- 向量相似度计算：余弦相似度、内积
- 语义缓存优化：避免重复调用大模型

**视觉元素**：
- 语义搜索工作流程图
- 向量嵌入可视化（2D t-SNE投影）

## 第四部分：关键模块实现

### 幻灯片 10：语义缓存模块
**设计目标**：
- 避免重复计算相似问题
- 加速问答响应速度
- 降低大模型API调用成本

**技术实现**：
```python
# 语义相似度计算
class _EmbeddingSimilarity:
    def similarity(self, text1, text2):
        # sentence-transformers生成嵌入
        embedding1 = self._model.encode(text1)
        embedding2 = self._model.encode(text2)
        # 余弦相似度
        return cosine_similarity(embedding1, embedding2)
```

**回退策略**：
- 优先使用sentence-transformers（高精度）
- 备选方案：字符n-gram相似度（零依赖）

**性能指标**：
- 缓存命中率：~40%
- 响应时间提升：3-5倍

- 语义缓存工作流程图
- 性能对比图表

### 幻灯片 11：多智能体协同系统
**LangGraph DAG流水线**：

```
输入文本
    ↓
[提取Agent] → 结构化JSON
    ↓
[审核Agent] → 校验通过/驳回
    ↓
[入库Agent] → Cypher写入Neo4j
```

**并发优化**：
- 多线程并发（ThreadPoolExecutor）

**状态管理**：
- HuxiangState：统一状态托盘
- 单向流转：确保数据处理严密性

### 幻灯片 16：工程挑战与解决方案
**关键挑战**：

1. **API限流（429错误）**
   - 解决：多线程 + 退避机制 + 错峰发车

2. **大模型超时**
   - 解决：放宽timeout至120-300秒

3. **JSON解析失败**
   - 解决：Pydantic Schema + 字符串容错清洗

4. **UNION语法陷阱**
   - 解决：提示词工程强制约束

5. **浏览器翻译干扰**
   - 解决：禁用自动翻译
