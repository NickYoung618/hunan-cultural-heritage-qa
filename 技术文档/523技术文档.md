# 湖湘文化 GraphRAG 多智能体系统 —— v2.2 架构优化与灰度测试实录

> **版本**: v2.2-Beta  
> **日期**: 2026-05-23  
> **状态**: 关系属性扩展（`detail`）完美成功；节点属性扩展（`description`）与多跳推理机制处于 Debug 阶段。

---

## 1. 架构升级背景

在前期单维语料（《王夫之》）的测试中，系统暴露出了“长文本评价被丢弃”、“孤立哲学概念无解释”以及“查询语句缺乏泛化能力”三大痛点。本次 v2.2 版本的核心目标，是将图谱从单一的“名词网络”升维为“承载详细历史文献的富属性网络（Property Graph）”。

---

## 2. 核心链路改造记录 (已执行)

通过协同 Claude Code，系统底层完成了以下维度的结构性重构：

### 2.1 抽取引擎升维 (`tools/extractor.py`)
* **关系扩展**：在 Pydantic 的 `Relationship` 模型中新增 `detail` 字段，专门承载评价原话。
* **节点扩展**：在 `Entity` 模型中新增 `description` 字段，用于捕捉孤立的哲学概念（如“气一元论”）的详细释义。
* **指令强化**：系统提示词（System Prompt）全面升级，强制大模型必须在抽取实体和关系时，将对应的原话摘抄进上述两个新字段。

### 2.2 数据库写入层改造 (`tools/graph_writer.py`)
* **属性持久化**：重写 Cypher 写入逻辑。在 `MERGE` 关系后新增 `SET r.detail = $detail`；在 `MERGE` 节点后新增 `SET n.description = $description`。

### 2.3 检索大脑重构 (`rag_agent.py`)
* **放弃精确匹配**：废除 `=` 语法，全面引入 `CONTAINS` 模糊匹配机制。
* **引入宽泛路径**：针对代词查询，引入 `[*1..2]` 多跳检索指令。
* **双重证据断言**：强制大模型在最终答案生成时，必须直接引用图谱返回的 `description` 或 `detail`。

## 3. 灰度测试验收报告 (真实战况)

本次测试使用了极为严苛的盲测标准，结果呈现出明显的“两极分化”状态，暴露出系统不同模块的稳健度差异。

### 🏆 Tier 1：完美成功的战役
* **防噪压制测试**：查询现代演员“陈昭荣”，系统成功拦截并回复无数据。证明 Prompt 的噪音过滤机制绝对生效。
* **直接关系长文本提取**：查询“曾国藩对王夫之的评价”，系统不仅精准寻路，更完整输出了长达数千字的文言文原话，并给出了高度专业的史学解读（耗时 41.85 秒）。
    * *结论*：**`Relationship.detail` 数据管道 100% 贯通，幻觉机制被彻底压制。**

### ⚠️ Tier 2：折戟沉沙的盲区（已知 Bug）
* **孤立节点失忆（概念测试）**：查询“气一元论”，系统依然交白卷。
    * *病灶分析*：虽然 `extractor.py` 已要求提取 `description`，但大模型在长文本切片抽取时，可能依然没有把几百字的哲学解释填入 `description` 字段；或者 `rag_agent.py` 生成的 Cypher 没有把这个孤立节点查出来。
* **多跳推理断链（家族树测试）**：查询“第四个儿子（王敔）”，系统交白卷。
    * *病灶分析*：即便启用了 `[*1..2]` 模糊检索，但自然语言转化为 Cypher 时，大模型极可能依然错误地将“第四个儿子”当做了强约束条件（例如 `WHERE b.name CONTAINS '第四个儿子'`），导致数据库无法将名为“王敔”的节点召回。家族关系的 Schema 未做标准化（抽取出的 relation 可能是“儿子”、“子嗣”、“四子”等，混乱不堪）。

## 4. 下一步行动蓝图 (v2.3 Debug 指南)

为了攻克剩下的两座堡垒，必须从“黑盒观测”转入“白盒调试”：

1.  **数据库探伤（查库验尸）**：
    * 必须直接在 Neo4j 浏览器中运行 `MATCH (n) WHERE n.name CONTAINS '王敔' RETURN n`，检查该节点是否存在，且是否与王夫之有连线。
    * 运行 `MATCH (n) WHERE n.name CONTAINS '一元论' RETURN n.description`，检查哲学释义是否真正落盘。
2.  **Cypher 生成截获**：
    * 修改 `rag_agent.py`，将 Text-to-Cypher 阶段大模型生成的原生 Cypher 语句强制 `print` 到终端。必须亲眼看到它是如何把“第四个儿子”翻译成代码的，才能精准修改提示词。
3.  **家族/学派关系标准化 (Schema Normalization)**：
    * 在 `extractor.py` 中增加强枚举约束，规定亲属关系只能使用统一的词汇（如：`父子`、`兄弟`、`夫妻`），拒绝大模型自由发挥，降低图谱遍历的复杂度。
